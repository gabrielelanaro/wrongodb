# Documentation

Maintained docs live in the following places:

- [`architecture/overview.md`](architecture/overview.md): current system structure and layer boundaries
- [`architecture/storage-engine.md`](architecture/storage-engine.md): storage metadata, B+tree/page/block internals, checkpointing, WAL, and recovery
- [`architecture/server-stack.md`](architecture/server-stack.md): MongoDB server layering, durable catalog, read path, and write path
- [`architecture/command-query-capabilities.md`](architecture/command-query-capabilities.md): implemented wire-protocol commands, query behavior, and current limitations
- [`architecture/guidelines.md`](architecture/guidelines.md): maintenance rules for architecture docs
- [`decisions.md`](decisions.md): dated record of non-trivial design decisions

Everything else is secondary:

- `NOTES.md` for investigations
- `notebooks/` for research
- `blog/` for historical narrative
